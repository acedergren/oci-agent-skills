# OCI PPTX

Use this when the deliverable is a presentation rather than an implementation artifact.

## ⚠️ Presentation Knowledge Gap

Architecture content and slide content are not the same thing. Strong OCI technical work still fails in review if it is dumped into generic cloud slides without Oracle framing, audience fit, or a crisp story arc.

## NEVER Do This

- Never dump a raw architecture diagram onto a slide and call it finished.
- Never use generic cloud branding when the deck is meant to represent Oracle and OCI.
- Never skip the audience decision between executive, technical buyer, and implementation team.
- Never lead with service sprawl when the audience needs business or operational impact first.
- Never turn migration risk into an unreadable wall of text.
- Never separate cost, security, and operations proof from the architecture story.

## Deck Order

1. Audience and decision the deck should support
2. One-slide architecture story
3. Security, cost, and operations proof
4. Migration or rollout sequence
5. Clear ask or next-step decision

## Typical Outputs

- Executive summary deck
- Technical design review deck
- CloudWorld-style product or solution pitch
- Migration proposal with phased rollout
